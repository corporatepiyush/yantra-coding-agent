# ytdl — YouTube & media downloader (yt-dlp)

YouTube (and 1000+ sites) media downloader via yt-dlp: read metadata (info/search) then download (download/audio/subtitles/transcript). URLs are http(s)-only and SSRF-checked, output is confined to `downloads/`, playlists are capped, and every download is consent-gated. Run `ytdl_doctor` first (needs yt-dlp; ffmpeg for audio/merges).

How to read this page:

- **Tool `name`** is the exact string for `tools/call`; **Workflow `name`** is `wf__<id>`.
- **Consent = yes** means the call changes state — over MCP it triggers an elicitation (or launch the server with `-y` to pre-approve). Consent = no calls are read-only.
- **LLM = mid/high** means the call sends content to a configured language model; `—` calls are fully local and free.
- If a call errors with "tool category disabled", enable it: `enable_category {"category":"ytdl"}` (or launch with `--enable ytdl`).
