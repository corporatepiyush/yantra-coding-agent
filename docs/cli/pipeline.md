# pipeline — pipeline

Composite quality gates: `pipeline.ci` = format + lint + build + test in one shot; preflight and fix variants.

How to read this page:

- **Tool `name`** is the exact string for `tools/call`; **Workflow `name`** is `wf__<id>`.
- **Consent = yes** means the workflow changes state — over MCP it triggers an elicitation (or launch the server with `-y`).
- **LLM = mid/high** means the call sends content to a configured language model; `—` calls are fully local and free.
- If a call errors with "tool category disabled", enable it: `enable_category {"category":"pipeline"}` (or launch with `--enable pipeline`).

## Workflows

Workflows chain several tools into one action. Over MCP they are tools named `wf__<id>` (dots become underscores). Pass inputs in `arguments`.

| Workflow `name` | Needs | Consent | What it does |
|-----------------|-------|---------|-------------|
| `wf__pipeline_ci` | — | no | Format+lint+build+test (pre-push gauntlet) |
| `wf__pipeline_fix` | — | **yes** | Format + auto-fix lint, then test |
| `wf__pipeline_preflight` | — | no | Read-only PR readiness: lint+test+secrets |

## Examples
