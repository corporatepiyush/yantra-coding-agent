# media — Media

Audio/video/image toolbox (ffmpeg-based): probe codecs first, then trim/convert/resize/concat/watermark/transcribe/strip-metadata.

How to read this page:

- **Tool `name`** is the exact string for `tools/call`; **Workflow `name`** is `wf__<id>`.
- **Consent = yes** means the call changes state — over MCP it triggers an elicitation (or launch the server with `-y` to pre-approve). Consent = no calls are read-only.
- **LLM = mid/high** means the call sends content to a configured language model; `—` calls are fully local and free.
- If a call errors with "tool category disabled", enable it: `enable_category {"category":"media"}` (or launch with `--enable media`).

## Workflows

Workflows chain several tools into one action. Over MCP they are tools named `wf__<id>` (dots become underscores). Pass inputs in `arguments`.

| Workflow `name` | Needs | Consent | What it does |
|-----------------|-------|---------|-------------|
| `wf__media_audiobook` | — | **yes** | Read a document aloud into one audio file (say / espeak-ng) |
| `wf__media_clip` | yt-dlp | **yes** | Download only a time range of a video (optionally shrink it) |
| `wf__media_hardsub` | ffmpeg | **yes** | Burn captions permanently into a local video (.srt or whisper) |
| `wf__media_podcast` | yt-dlp | **yes** | Talk URL -> loudness-normalized, tagged offline audio |
| `wf__media_share_photos` | — | **yes** | Strip EXIF, downscale, optionally blur faces, and bundle photos to share |
| `wf__media_summarize` | — | no | Transcribe a video/talk (URL or file) and summarize it |
