# git — Git (read-only introspection)

Read-only git introspection: history, diffs, show, "who introduced this string?" (pickaxe), per-file history, ahead/behind. Mutations (commit/push/branch) live in the `git.*` workflows below, behind consent. (Plain status/branches/contributors are one `bash` call — use that.)

How to read this page:

- **Tool `name`** is the exact string for `tools/call`; **Workflow `name`** is `wf__<id>`.
- **Consent = yes** means the call changes state — over MCP it triggers an elicitation (or launch the server with `-y` to pre-approve). Consent = no calls are read-only.
- **LLM = mid/high** means the call sends content to a configured language model; `—` calls are fully local and free.
- If a call errors with "tool category disabled", enable it: `enable_category {"category":"git"}` (or launch with `--enable git`).

## Workflows

Workflows chain several tools into one action. Over MCP they are tools named `wf__<id>` (dots become underscores). Pass inputs in `arguments`.

| Workflow `name` | Needs | Consent | What it does |
|-----------------|-------|---------|-------------|
| `wf__git_bisect` | git | no | Bisect for regression |
| `wf__git_blame-line` | git | no | Show blame for a line |
| `wf__git_branch` | git | no | List/create/switch branches |
| `wf__git_clean` | git | no | Show untracked files |
| `wf__git_commit` | git | **yes** | Commit staged changes |
| `wf__git_conflict-assist` | git | **yes** | Show both sides of a conflict, then gated --continue/--abort |
| `wf__git_pr` | git gh | **yes** | Open PR with a vetted title+body (via pr.prepare) |
| `wf__git_quicksave` | git | **yes** | Add+commit+push |
| `wf__git_release` | git | **yes** | Tag + push a release (validated, gated, honest rc) |
| `wf__git_rescue` | git | no | Lost-work first aid: reflog+stashes+dangling commits+recovery recipes |
| `wf__git_ship` | git | **yes** | Honest release pipeline: preflight+validate+tag+push+gh release |
| `wf__git_stash` | git | no | Stash current changes |
| `wf__git_sync` | git | **yes** | Fetch+rebase+push |
| `wf__git_undo` | git | **yes** | Undo last commit (soft reset) |
| `wf__git_worktree` | git | **yes** | List/add/remove/prune worktrees |
