# s3 — S3 Storage

S3-compatible object storage (SigV4 via env creds): upload/download/list/delete/head.

How to read this page:

- **Tool `name`** is the exact string for `tools/call`; **Workflow `name`** is `wf__<id>`.
- **Consent = yes** means the call changes state — over MCP it triggers an elicitation (or launch the server with `-y` to pre-approve). Consent = no calls are read-only.
- **LLM = mid/high** means the call sends content to a configured language model; `—` calls are fully local and free.
- If a call errors with "tool category disabled", enable it: `enable_category {"category":"s3"}` (or launch with `--enable s3`).

## Examples

{"method":"tools/call","params":{"name":"s3_presign","arguments":{"key":"KEY"}}}

{"method":"tools/call","params":{"name":"s3_download","arguments":{"key":"KEY"}}}   // needs consent
{"method":"tools/call","params":{"name":"s3_sync","arguments":{"dir":"VALUE"}}}   // needs consent
