# docker — Docker

Container operations: ps/logs/inspect/diff/health, build/run/exec/cp/push/restart/prune (gated) — plus LLM triage ("why is my container crashing?"). The `container.overview` workflow is a one-call ps+stats+health+reclaimable snapshot.

How to read this page:

- **Tool `name`** is the exact string for `tools/call`; **Workflow `name`** is `wf__<id>`.
- **Consent = yes** means the call changes state — over MCP it triggers an elicitation (or launch the server with `-y` to pre-approve). Consent = no calls are read-only.
- **LLM = mid/high** means the call sends content to a configured language model; `—` calls are fully local and free.
- If a call errors with "tool category disabled", enable it: `enable_category {"category":"docker"}` (or launch with `--enable docker`).
