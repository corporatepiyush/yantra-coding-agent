# data — Data (duckdb)

SQL over data files with DuckDB — CSV/Parquet/Arrow, no import step: query, schema, head, profile, join, convert — plus LLM insights.

How to read this page:

- **Tool `name`** is the exact string for `tools/call`; **Workflow `name`** is `wf__<id>`.
- **Consent = yes** means the call changes state — over MCP it triggers an elicitation (or launch the server with `-y` to pre-approve). Consent = no calls are read-only.
- **LLM = mid/high** means the call sends content to a configured language model; `—` calls are fully local and free.
- If a call errors with "tool category disabled", enable it: `enable_category {"category":"data"}` (or launch with `--enable data`).

## Workflows

Workflows chain several tools into one action. Over MCP they are tools named `wf__<id>` (dots become underscores). Pass inputs in `arguments`.

| Workflow `name` | Needs | Consent | What it does |
|-----------------|-------|---------|-------------|
| `wf__data_convert` | duckdb | **yes** | Convert data format (csv/parquet/json) |
| `wf__data_diff` | duckdb | no | Diff two data files: schema, row counts, sample + keyed changes |
| `wf__data_join` | duckdb | no | Join two data files on a shared column |
| `wf__data_profile` | duckdb | no | Profile data (stats, nulls, types) |
| `wf__data_query` | duckdb | no | Query a data file (SQL over the table 'this') |
| `wf__data_summary` | duckdb | no | Summarize a data file (schema) |
