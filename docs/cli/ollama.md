# ollama — Ollama / Local LLM

Drive local LLMs via Ollama: one-shot run, model-file/notebook inspection, embeddings + RAG (DuckDB vector store), chat/generate with option merging, structured extract, serve status/disk/logs. (Plain list/pull/ps/rm are one `bash` call — use that.)

How to read this page:

- **Tool `name`** is the exact string for `tools/call`; **Workflow `name`** is `wf__<id>`.
- **Consent = yes** means the call changes state — over MCP it triggers an elicitation (or launch the server with `-y` to pre-approve). Consent = no calls are read-only.
- **LLM = mid/high** means the call sends content to a configured language model; `—` calls are fully local and free.
- If a call errors with "tool category disabled", enable it: `enable_category {"category":"ollama"}` (or launch with `--enable ollama`).
