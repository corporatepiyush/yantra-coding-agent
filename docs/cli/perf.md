# perf — Performance

System and process profiling: CPU/memory/IO/network load, top processes, open files, strace/dtrace, benchmarks, binary size.

How to read this page:

- **Tool `name`** is the exact string for `tools/call`; **Workflow `name`** is `wf__<id>`.
- **Consent = yes** means the call changes state — over MCP it triggers an elicitation (or launch the server with `-y` to pre-approve). Consent = no calls are read-only.
- **LLM = mid/high** means the call sends content to a configured language model; `—` calls are fully local and free.
- If a call errors with "tool category disabled", enable it: `enable_category {"category":"perf"}` (or launch with `--enable perf`).

## Workflows

Workflows chain several tools into one action. Over MCP they are tools named `wf__<id>` (dots become underscores). Pass inputs in `arguments`.

| Workflow `name` | Needs | Consent | What it does |
|-----------------|-------|---------|-------------|
| `wf__perf_baseline` | — | **yes** | Time build+test, save as baseline (.yca-perf-baseline.json) |
| `wf__perf_compare` | — | no | Rerun build+test and flag drift >15% vs baseline |
