# deps — deps

Dependency lifecycle: install, update, audit, license check, tree — and `deps.risk`: outdated + CVEs + loose version specs with a senior upgrade order.

How to read this page:

- **Tool `name`** is the exact string for `tools/call`; **Workflow `name`** is `wf__<id>`.
- **Consent = yes** means the workflow changes state — over MCP it triggers an elicitation (or launch the server with `-y`).
- **LLM = mid/high** means the call sends content to a configured language model; `—` calls are fully local and free.
- If a call errors with "tool category disabled", enable it: `enable_category {"category":"deps"}` (or launch with `--enable deps`).

## Workflows

Workflows chain several tools into one action. Over MCP they are tools named `wf__<id>` (dots become underscores). Pass inputs in `arguments`.

| Workflow `name` | Needs | Consent | What it does |
|-----------------|-------|---------|-------------|
| `wf__deps_add` | — | **yes** | Add a dependency (toolchain-aware) |
| `wf__deps_audit` | — | no | Security audit dependencies |
| `wf__deps_install` | — | **yes** | Install dependencies |
| `wf__deps_licenses` | — | no | Check license compliance |
| `wf__deps_risk` | — | no | Outdated+audit+loose specs with a senior upgrade order |
| `wf__deps_tree` | — | no | Show dependency tree |
| `wf__deps_update` | — | **yes** | Update dependencies |
