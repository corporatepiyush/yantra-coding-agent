# net — Network

Network triage from the shell: DNS, traceroute, port checks, TLS cert expiry, HTTP headers, and `net_fetch` (SSRF-vetted, fenced download of any URL to a file) — or run the `net.diagnose` workflow for the full DNS→TCP→TLS→HTTP sweep, or `net.watch` to be told when a page changes.

How to read this page:

- **Tool `name`** is the exact string for `tools/call`; **Workflow `name`** is `wf__<id>`.
- **Consent = yes** means the call changes state — over MCP it triggers an elicitation (or launch the server with `-y` to pre-approve). Consent = no calls are read-only.
- **LLM = mid/high** means the call sends content to a configured language model; `—` calls are fully local and free.
- If a call errors with "tool category disabled", enable it: `enable_category {"category":"net"}` (or launch with `--enable net`).

## Workflows

Workflows chain several tools into one action. Over MCP they are tools named `wf__<id>` (dots become underscores). Pass inputs in `arguments`.

| Workflow `name` | Needs | Consent | What it does |
|-----------------|-------|---------|-------------|
| `wf__net_diagnose` | curl | no | DNS→TCP→TLS→HTTP diagnosis for a URL or host |
| `wf__net_watch` | curl | **yes** | Snapshot a page and report when it changes (line diff) |
